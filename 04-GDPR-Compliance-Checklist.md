# GDPR Compliance Checklist

This checklist provides a structured set of GDPR compliance controls spanning governance, technical security, data subject rights, and breach response.  
It is designed to support internal reviews, audits, and privacy-by-design workflows.

---

## 1. Data Governance & Documentation
- [ ] Data inventory updated and accurate  
- [ ] ROPA created and reviewed regularly  
- [ ] Lawful basis documented for each processing activity  
- [ ] DPIA completed for all high-risk processing  
- [ ] Retention schedules defined and aligned with purpose limitation  

---

## 2. Technical Security Controls
- [ ] Encryption in transit (TLS 1.2/1.3)  
- [ ] Encryption at rest enabled  
- [ ] Access controls enforce least privilege  
- [ ] Logs monitored for unauthorized access  
- [ ] Data backups encrypted and tested  

---

## 3. Privacy by Design
- [ ] Data minimization principles applied  
- [ ] Purpose limitation enforced at collection and processing  
- [ ] Pseudonymization or anonymization applied where appropriate  
- [ ] Systems undergo privacy/security review before deployment  

---

## 4. Data Subject Rights (DSAR)
- [ ] DSAR workflow documented  
- [ ] Identity verification steps defined  
- [ ] Ability to execute rights: access, rectification, erasure, portability, restriction  
- [ ] DSAR log maintained  

---

## 5. Incident Response & Breach Reporting
- [ ] Breach response plan documented  
- [ ] 72-hour reporting workflow tested  
- [ ] Team roles (DPO, Security, Legal) defined  
- [ ] Evidence preservation steps documented  

---

## 6. Vendor & Third-Party Risk
- [ ] Data Processing Agreements (DPAs) in place  
- [ ] Vendor security reviews performed annually  
- [ ] Sub-processors identified and monitored  
- [ ] Cross-border processing assessed and documented  

---

## Notes
This checklist aligns with GDPR principles and can be used as part of privacy readiness assessments, internal audits, or ongoing governance activities.
